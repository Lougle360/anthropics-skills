阶段四操作手册

Web 开发与测试技能详解

第一部分：深度知识点

1. React 组件化架构

组件设计原则

  • 单一职责：每个组件只做一件事
  • 可复用性：组件应该可以在多处使用
  • 组合性：小组件组合成大组件
  • 可测试性：组件应该容易测试

Hooks 最佳实践

  • useState：简单状态管理
  • useEffect：副作用和生命周期
  • useContext：跨组件状态共享
  • useCallback/useMemo：性能优化

TypeScript 类型安全

interface User { id: string; name: string; email: string; } interface Props { user: User; onUpdate: (user: User) => void; } function UserProfile({ user, onUpdate }: Props) { const [editing, setEditing] = useState(false); // 组件逻辑 }

2. Playwright 测试架构

页面对象模式

封装页面元素的类，提供清晰的 API。

class LoginPage { constructor(page: Page) { this.page = page; this.usernameInput = page.locator('#username'); this.passwordInput = page.locator('#password'); this.loginButton = page.locator('button[type="submit"]'); } async login(username: string, password: string) { await this.usernameInput.fill(username); await this.passwordInput.fill(password); await this.loginButton.click(); await this.page.waitForURL('/dashboard'); } } // 使用 test('user can login', async ({ page }) => { const login = new LoginPage(page); await login.goto('/'); await login.login('test@example.com', 'password'); });

测试金字塔理论

  • E2E 测试：少量，覆盖关键用户流程
  • 集成测试：中等数量，测试组件交互
  • 单元测试：大量，测试独立函数

3. 响应式设计

Flexbox 布局

.container { display: flex; justify-content: space-between; align-items: center; } .item { flex: 1; margin: 0 10px; }

CSS Grid 系统

.grid { display: grid; grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); gap: 20px; } .card { grid-column: span 2; /* 跨两列 */ }

第二部分：操作指南

1. React 数据可视化仪表板完整实现

  1. 设置项目：初始化 React + TypeScript 项目
  2. 安装依赖：Tailwind、图表库、shadcn/ui
  3. 设计组件结构：Dashboard、Chart、DataTable、FilterPanel
  4. 实现组件：
    • Dashboard：主容器，布局
    • Chart：数据可视化（使用 Chart.js 或 Recharts）
    • DataTable：数据表格（排序、筛选、分页）
    • FilterPanel：筛选控制
  5. 添加状态管理：使用 Context API 或 Redux
  6. 实现响应式布局：移动、平板、桌面
  7. 应用主题和样式
  8. 添加加载状态和错误处理
  9. 优化性能：懒加载、虚拟滚动
提示：使用 TypeScript 严格模式，确保所有类型正确定义。

2. Playwright 测试套件完整流程

  1. 设置测试环境：初始化 Playwright 项目
  2. 设计测试架构：页面对象、测试工具
  3. 编写测试用例：
    • 正向测试：预期成功的操作
    • 负向测试：错误处理
    • 边界测试：极限情况
    • UI 测试：视觉验证
  4. 实现页面对象：
    • 为每个主要页面创建类
    • 封装常见元素和操作
    • 添加等待和断言
  5. 配置测试运行器：
    • 设置超时和重试
    • 配置测试报告
    • 设置截图失败
  6. 集成 CI/CD：GitHub Actions、GitLab CI
  7. 运行和审查结果

3. 响应式页面设计流程

  1. 设计阶段：
    • 创建线框图（三种设备尺寸）
    • 设计高保真原型
    • 确定断点（移动、平板、桌面）
  2. 实现阶段：
    • 使用移动优先方法
    • 实现 Flexbox 和 Grid 布局
    • 应用响应式图片和媒体
    • 使用相对单位（rem、%、vh/vw）
  3. 测试阶段：
    • 在实际设备上测试
    • 使用浏览器开发工具
    • 检查可访问性（屏幕阅读器、键盘导航）
  4. 优化阶段：
    • 压缩图片
    • 最小化 CSS 和 JS
    • 实施懒加载
    • 缓存策略
